# 端到端真机测试清单

本文档用于在手机 Safari 上验证 AI 视觉对话助手的完整演示链路。桌面浏览器只能验证布局和无权限路径，摄像头、麦克风、语音识别、TTS 自动播放和蜂窝网络表现必须用真机复核。

## 测试前准备

- 本地 `.env` 已配置 `AI_API_KEY`，后端 `/api/health` 返回 `providerConfigured: true`。
- 使用 `npm run dev` 启动前端和后端代理。
- 使用 `scripts/start-ngrok.ps1` 暴露 Vite 前端地址，并在手机 Safari 打开 ngrok HTTPS 链接。
- 手机系统已允许 Safari 访问摄像头和麦克风。
- 关闭系统低电量模式；测试弱网时再单独打开网络限制或切换蜂窝网络。

## 通过标准

- P0 链路必须通过：摄像头授权、语音提问、自动抓帧、AI 回答、TTS 朗读。
- P1 体验必须通过：首屏引导清晰，AI 思考时有主视觉反馈。
- P2 兜底必须通过：文本输入能在语音不可用时完成提问。
- P3 增强项至少通过核心路径：自动追问、弱网慢响应提示、失败重试、低清/高清模式、回答复制。
- 所有失败状态必须给出下一步动作，不允许只显示技术错误。
